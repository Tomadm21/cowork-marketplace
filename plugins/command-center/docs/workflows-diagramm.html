⚙️ Command Center — so laufen die Workflows

Ein Eingang, fünf Prozesse, ein Review-Board, eine Engine. Nichts wird ohne deine Freigabe gespeichert.
Plugin v0.9.0 · Stand 02.07.2026 · inkl. Bericht-Scans (Modus B)

Der Alltagsweg — „verarbeite alles"

📥  Ein gemeinsamer Eingang: _eingang/
Alles einfach reinwerfen — gemischt: Fotos, Belege, Bericht-Scans, ACHIM-Rapporte, Stundenzettel.
Auch externe Eingänge möglich (intake.json: Ordner-Routing, z. B. Netzwerk-Ordner).
🔍  Intake — erkennen, entdoppeln, zuordnen
Jede Datei wird nach ihrem Inhalt klassifiziert, nie nach Endung oder Ordner.
1 · md5-Dublettencheck (auch gegen früher Abgelegtes) 2 · Magic-Bytes: „PDF“ das ein Foto ist? → als Bild lesen 3 · Vision/Text-Klassifikation je Datei 4 · Notiz/unklar → Sammelfrage am Ende
🗂️ Belege ablegen
Rechnung · Lieferschein · Gutschrift · Quittung
  1. Auslesen: Lieferant, Nr., Datum, Betrag, Typ (RG/LF/LS)
  2. Firma ableiten (Adressat ↔ company-context)
  3. Namensschema + Ablageordner vorschlagen
  4. SEPA/offen? → nur als „prüfen“ markieren, nie fragen
→ Beleg richtig benannt im Buchhaltungs-Ordner
📷 Fotos sortieren (Modus A)
WhatsApp-/Handy-Bilder von der Baustelle
  1. Datum: Dateiname/EXIF · Morgenbild (7–11 Uhr) = Vortag
  2. Baustelle: Abgleich stammdaten/projekte.json
  3. Tätigkeit wörtlich aus dem Bautagesbericht — Katalog nur als Fallback · Inhalt schlägt Zeitstempel
  4. Name: Datum_Baustelle_Tätigkeit_Nr
→ Projekt-/Bilder-Ordner, optional KW-Unterordner
📄 Bericht-ScansNEU (Modus B)
abfotografierte handschriftliche Montage-/Serviceberichte
  1. Vision liest den Bericht: Datum-Spalten → Jahr+KW (So-Anreise = laufende KW)
  2. BV-Kurzform (Ort + Kunde) aus dem Kopf
  3. Monteure in exakter Schreibweise aus stammdaten/monteure.json — unklare Handschrift → fragen, nie raten
  4. Name: JJJJ KWnn BV V.Nachname … (+ _Anreise/_Nachtrag)
→ Berichte-Ordner / KWnn/
↔ Doppel-Route: enthält der Scan Abrechnungs-Stunden, läuft zusätzlich „Rechnung“ mit derselben Quelle
📋 Tagesbericht
ACHIM-Rapport / Regiebericht (Rohdaten)
  1. Auslesen: Projekt, KW, Personen, Von/Bis je Tag
  2. Netto-Stunden berechnen (Tages-Cap, transparent)
  3. Firmen-Vorlage füllen (DOCX)
  4. Gecappte Tage → „prüfen“
→ fertiger Bautagesbericht in _ausgang/berichte — wird wieder Quelle für die Foto-Tätigkeiten!
🧾 Rechnung
Stundenzettel / Wochenstunden zur Abrechnung
  1. Stunden je Arbeiter auslesen (Woche)
  2. Pflicht-Skript compute.ts rechnet Sätze, Spesen, MwSt — nie Kopfrechnen
  3. Pro-forma-Rechnung (XLSX) bauen
→ prüffertige Rechnung in _ausgang/rechnungen
🗃️  Vorbereitet — noch nichts bewegt
Je Prozess eine Review-Queue _firma/_review/R-‹Datum›-‹Prozess›.json + Activity-Log prepared.
Jeder Posten trägt eine Stufe:
sicher prüfen folgenreich
📋  Review-Board — ein Prozess nach dem anderen
Reihenfolge: Belege → Fotos/Scans → Tagesbericht → Rechnung. Pro Posten:
🪪 Editierbare Karte
Dateiname, Speicherort, alle Werte (z. B. Betrag · Tätigkeit · Jahr/KW/BV/Monteure), Begründung, Stufen-Badge — Buttons „Übernehmen“ / „Ablehnen“
📄 + 📎 Datei-Boxen darunter
Ergebnis (Vorschau unter Zielnamen) und Quelle (Original) — Klick öffnet nativ rechts in der Cowork-Sidebar
✅ Freigeben — ‹Prozess› speichern (N)
… speichert diesen Prozess und zeigt sofort den nächsten.
⚙️  Apply-Engine _firma/apply.py
Kopiert — Original bleibt immer unangetastet. Kollisionssicher (_2 statt überschreiben), md5-idempotent (identische Datei wird nie doppelt geklont), atomar.
Journal-Zeile je Ablage → umkehrbar Queue leer → Archiv _erledigt/ Activity-Log → done
🖥️ N:/S: verbunden
direkt in die echten Firmen-Ordner
📁 sonst _ausgang/
+ vorgesehener N:-Pfad wird notiert
📊 Dashboard
Live-Artefakt: Stunden gespart, erledigte Arbeit, nächster Schritt

Läuft nebenher

🏢 Firmen-Onboarding (einmalig)
Interview → company-context.md + Workspace-Gerüst + Engine + Eingang-Mapping. Jeder Prozess erbt den Kontext und onboardet sich beim ersten Lauf selbst (detect-first, nummerierte Optionen + ✏️).
⏰ Sammel-Modus (stündlich)
Gleicher Ablauf bis „vorbereitet“, aber ohne Rückfragen und ohne Board — Unklares wird „prüfen“. Kurze Notiz: „X Posten liegen zur Freigabe.“ Gespeichert wird nie unbeaufsichtigt.
🔁 Lernschleife
Neue Baustelle / neuer Lieferant / neuer Monteur → „prüfen“ + Fakt-Signal. Nach Freigabe einmalige Frage: „In Stammdaten übernehmen?“ → derselbe Fall ist künftig sicher.

Die 5 Sicherheits-Prinzipien

🔒
Copy-only
Originale werden nie umbenannt oder verschoben
↩️
Umkehrbar
Jede Ablage steht als Journal-Zeile im _journal/
🚪
Freigabe-Gate
Gespeichert wird nur auf deinen Klick — nie im Hintergrund
🧬
Keine Dubletten
md5 über Eingang, Ziel und Historie
🏳️
Firmen-neutral
Alle Firmendaten in stammdaten/ + Config — nichts hardcoded
